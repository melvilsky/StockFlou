# Аудит дизайна StockFlou

## 1. Текущая цветовая система

| Элемент | Светлая тема | Тёмная тема |
|--------|--------------|-------------|
| **Primary** | `#137FEC` | тот же |
| **Фон приложения** | `#F6F7F8` | `#101922` |
| **Панели (surface)** | белый | `#0F172A` (slate-900) |
| **Текст** | `#0F172A` (slate-900) | `#F1F5F9` (slate-100) |
| **Границы** | `#E2E8F0` (slate-200) | `#1E293B` (slate-800) |

**Плюсы:** Единый primary, осмысленные нейтральные оттенки (slate), Material 3, Inter — хорошая база.

## 2. Найденные проблемы

### 2.1 Контраст и доступность (a11y)

- **History screen — бейдж "Total Progress: 45%"**  
  Используется `onSurface.withValues(alpha: 0.2)` — контраст слишком низкий, текст почти не читается. Нужно минимум ~0.5–0.6 для второстепенного текста.

- **Мелкий текст (11–12 px)** в подписях и бейджах при alpha 0.5 может не достигать 4.5:1 на части фонов. Рекомендуется проверить в DevTools и при необходимости поднять до 0.6.

### 2.2 Тёмная тема

- **History screen — кнопки "Pause" и текст**  
  Жёстко заданы `Colors.black87` и `Colors.black` — в тёмной теме кнопка и текст сливаются с фоном. Нужно брать цвет из темы (`colorScheme.onSurface` и т.п.).

### 2.3 Жёстко заданные цвета (не из темы)

- Много мест используют `Colors.green`, `Colors.red`, `Colors.blue`, `Colors.amber`, `Colors.orange`, `Colors.grey` без привязки к теме. В тёмной теме оттенки могут быть слишком яркими или плохо сочетаться с фоном.
- **Рекомендация:** ввести в `AppTheme` семантические цвета (success, error, warning) и использовать их в SnackBar, бейджах, иконках статуса.

### 2.4 Несоответствие названия

- В `main.dart` в сайдбаре отображается **"StockFlow"**, в `MaterialApp.title` — **"StockFlou"**. Стоит унифицировать (либо везде StockFlou, либо StockFlow).

### 2.5 Визуальная иерархия

- Сайдбар и левая панель действий (72px) визуально очень похожи (surface + border). Можно чуть усилить различие (например, лёгкий фон у сайдбара или скругление только у контента), чтобы яснее разделять «навигацию» и «инструменты».

### 2.6 Интерактивность (по чек-листу UI UX Pro)

- **Hover:** используется `InkWell` — ок.
- **Курсор:** на десктопе у кнопок и `InkWell` по умолчанию обычно pointer — достаточно.
- **Переходы:** явных `duration` для hover нет — можно задать 150–300 ms для смены цвета/фона у карточек и пунктов сайдбара.

## 3. Рекомендации по улучшению

### 3.1 Цвета

1. **Добавить в `AppTheme` семантические цвета**  
   Например: `successColor`, `errorColor`, `warningColor` (и при желании варианты для фона/обводки), использовать их в SnackBar, статусах (Tagged/Untagged, успех/ошибка загрузки).
2. **Заменять все жёсткие `Colors.xxx`** на цвета из темы или `AppTheme`, чтобы светлая/тёмная тема и возможный будущий бренд-цвет работали предсказуемо.
3. **Не использовать alpha ниже 0.45** для текста, который должен быть читаемым; для второстепенного текста — ориентир 0.55–0.7.

### 3.2 Типографика

- Inter уже подключён — хорошо. Можно явно задать в `textTheme` размеры для заголовков, тела и подписей (например, `titleLarge`, `bodyMedium`, `labelSmall`) и использовать их по всему приложению для единообразия.

### 3.3 Компоненты

- **SnackBar:** один стиль успеха/ошибки через тему (например, `AppTheme.successColor` / `errorColor`) и общий `behavior: SnackBarBehavior.floating` для современного вида.
- **Кнопки:** везде уже скругление 8 — ок. Для опасных действий (Cancel, удаление) использовать один стиль (например, `errorColor` или `colorScheme.error`).
- **Карточки (History stat cards, Inspector):** тени очень мягкие — норм; можно добавить лёгкий `borderRadius` везде одинаковый (у вас в основном 8–12).

### 3.4 Тёмная тема

- После замены `black87`/`black` на `colorScheme.onSurface` и т.п. проверить все экраны в dark mode.
- Яркие акценты (зелёный успех, красный ошибка) в тёмной теме можно слегка приглушить (например, через `Color.lerp` с surface или готовые shade), чтобы не резало глаз.

## 4. Краткий чек-лист после правок

- [ ] Все тексты с достаточным контрастом (≥ 4.5:1 для основного текста).
- [ ] Нет жёстких `Colors.black`/`white` для текста и кнопок — только из темы.
- [ ] SnackBar и статусы используют цвета из `AppTheme` / ColorScheme.
- [ ] Единое название приложения (StockFlou vs StockFlow).
- [ ] В тёмной теме все экраны читаемы и кнопки заметны.
