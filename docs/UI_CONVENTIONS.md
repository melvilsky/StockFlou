# UI-конвенции и клики (StockFlou)

## Одиночный клик на desktop (macOS / Windows)

### В чём проблема

На desktop во Flutter элементы с `InkWell` или `GestureDetector(onTap: ...)` часто реагируют только на **двойной клик**. Один клик «съедается» gesture arena. Это известное поведение, его не исправляют заменой на другой стандартный виджет — нужен обход через pointer-события.

### Стандарт проекта

Для **любых областей, где по дизайну должен срабатывать один клик** (карточки в сетке, строки списков, кастомные кликабельные блоки), в проекте используется виджет:

**`lib/core/widgets/single_click_area.dart`** — `SingleClickArea`

Он обрабатывает pointer down/up через `Listener`, поэтому один клик срабатывает сразу и не ломается при следующих правках.

### Как использовать

1. Импорт (путь от текущего файла до `lib/core/widgets/single_click_area.dart`):
   ```dart
   import '../../../core/widgets/single_click_area.dart';
   ```

2. Оборачивать кликабельную зону в `SingleClickArea`, обработчик вешать на него:
   ```dart
   SingleClickArea(
     onTap: () => _onItemTap(item),
     child: Material(
       color: Colors.transparent,
       child: InkWell(
         onTap: () {}, // только рипл, без своего обработчика
         borderRadius: BorderRadius.circular(12),
         child: ...,
       ),
     ),
   )
   ```

3. **Не** заменять `SingleClickArea` на один только `InkWell(onTap: ...)` или `GestureDetector(onTap: ...)` в таких местах — иначе снова начнёт требоваться двойной клик.

### Где применять

- Карточки в сетках (превью файлов, медиа).
- Элементы списков с выбором по клику.
- Любые кастомные «тап-зоны», где по ТЗ должен срабатывать один клик.

Обычные кнопки (`ElevatedButton`, `IconButton`, `TextButton`) можно не оборачивать.

### Зачем это в инструкциях

Чтобы при рефакторинге или добавлении новых экранов не убирать `SingleClickArea` и не возвращать проблему «срабатывает только двойной клик». Использовать его везде, где нужен одиночный клик по области.
